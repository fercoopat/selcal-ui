# Component Patterns

## Core Principles

1. **All components are memoized** to prevent unnecessary re-renders.
2. **Barrel exports**: Every directory has an `index.ts` re-exporting its contents.
3. **Page components are thin**: They call hooks and pass data to presentational components.
4. **UI primitives live in `@/components/ui/`**: shadcn/ui components, never modified directly.

## Component Export Pattern

```tsx
// my-component.tsx
import { memo } from "react";

interface MyComponentProps {
  title: string;
  onAction: () => void;
}

const MyComponent = ({ title, onAction }: MyComponentProps) => {
  return (
    <div>
      <h2>{title}</h2>
      <button onClick={onAction}>Action</button>
    </div>
  );
};

export default memo(MyComponent);
```

### When to Use `genericMemo` HOC

Use the `genericMemo` HOC from `@/components/hoc/generic-memo` when the component needs to preserve generic type parameters:

```tsx
import { genericMemo } from "@/components/hoc/generic-memo";

interface DataTableProps<T> {
  data: T[];
  columns: ColumnDef<T>[];
}

const DataTable = <T,>({ data, columns }: DataTableProps<T>) => {
  // ...
};

export default genericMemo(DataTable);
```

## Page Component Pattern

Page components are thin wrappers that orchestrate hooks and delegate rendering:

```tsx
// modules/projects/pages/projects-list-page.tsx
import { memo } from "react";
import { useFindAllProjects } from "../hooks/use-find-all-projects";
import { DataTable } from "@/components/data-table";
import { projectsColumns } from "../constants/projects-list-columns";

const ProjectsListPage = () => {
  const { error, isLoading, projects } = useFindAllProjects();

  return (
    <DataTable
      columns={projectsColumns}
      data={projects}
      error={error}
      isLoading={isLoading}
    />
  );
};

export default memo(ProjectsListPage);
```

## Form Patterns

### FormContainer Wrapper

All forms use the `FormContainer` component which provides `FormProvider` + `<form>` with `handleSubmit`:

```tsx
import { FormContainer } from "@/components/forms/form-container";
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { createProjectSchema, type CreateProjectPayload } from "../schemas";

const CreateProjectForm = () => {
  const form = useForm<CreateProjectPayload>({
    resolver: zodResolver(createProjectSchema),
    defaultValues: {
      name: "",
      description: "",
    },
  });

  const onSubmit = form.handleSubmit(async (data) => {
    // mutation logic
  });

  return (
    <FormContainer form={form} onSubmit={onSubmit}>
      <FormFieldInput control={form.control} name="name" label="Project Name" />
      <FormFieldTextarea
        control={form.control}
        name="description"
        label="Description"
      />
      <LoadingButton type="submit" loading={form.formState.isSubmitting}>
        Create
      </LoadingButton>
    </FormContainer>
  );
};
```

### Form Field Components

Use the form field wrappers from `@/components/forms/`:
- `FormFieldInput` - Text inputs
- `FormFieldTextarea` - Textareas
- `FormFieldSelect` - Select dropdowns
- `FormFieldDate` - Date pickers
- `FormFieldMultiSelect` - Multi-select
- `FormFieldFiles` - File uploads

## Data Table Pattern

Use the `DataTable` component from `@/components/data-table/` for all tabular data:

```tsx
import { DataTable } from "@/components/data-table";
import { ColumnDef } from "@tanstack/react-table";

const columns: ColumnDef<Project>[] = [
  {
    accessorKey: "name",
    header: "Name",
    cell: ({ row }) => <span>{row.original.name}</span>,
  },
  {
    accessorKey: "createdAt",
    header: "Created",
    cell: ({ row }) => formatDate(row.original.createdAt),
  },
  {
    id: "actions",
    cell: ({ row }) => <RowActions project={row.original} />,
  },
];

<DataTable
  columns={columns}
  data={projects}
  isLoading={isLoading}
  error={error}
  onRetry={() => refetch()}
/>;
```

Column definitions live in module `constants/` files (e.g., `projects-list-columns.tsx`).

## shadcn/ui Components

All UI primitives are from shadcn/ui (New York style) in `@/components/ui/`:

- **Never modify** files in `@/components/ui/` directly. These are generated by the shadcn CLI.
- **Compose** primitives to build complex UI.
- **Extend** via wrapper components in `@/components/` if needed.

### Adding New shadcn Components

```bash
pnpm dlx shadcn@latest add <component-name>
```

## Layout Components

### Available Layouts
- `AuthLayout` - Login/register pages (centered, minimal chrome)
- `MainLayout` - Primary app layout with sidebar, header, content area
- `ModuleLayout` - Redirects from module root to default sub-route

### Usage

```tsx
// In route definitions
{
  path: "/projects",
  element: <MainLayout />,
  children: [
    { index: true, element: <ModuleLayout redirectTo="list" /> },
    { path: "list", element: <ProjectsListPage /> },
  ],
}
```

### Mutation Feedback
All mutations show toast notifications via `sonner`:
```tsx
import { toast } from "sonner";

const mutation = useMutation({
  mutationFn: createProject,
  onSuccess: () => {
    toast.success("Project created successfully");
    queryClient.invalidateQueries({ queryKey: PROJECTS_QUERIES.findAll });
  },
  onError: (error) => {
    toast.error(`Failed to create project: ${error.message}`);
  },
});
```

## Accessibility

- All interactive elements must have accessible labels (use `aria-label` or `aria-labelledby`).
- Form fields must have associated `<label>` elements (handled by `FormField*` components).
- Color is never the sole means of conveying information.
- Target WCAG 2.1 AA compliance for industrial use (tablets, workshop monitors).

## Performance Guidelines

1. **Memoize** all components by default.
2. **Lazy load** routes with `React.lazy()` + `Suspense` + `PageLoader`.
3. **Virtualize** large tables if needed (TODO: add virtualization for 1000+ rows).
4. **Debounce** search inputs with `useDebounce` hook.
5. **Avoid** inline object/array creation in JSX props.

```tsx
// Bad: creates new array every render
<MyComponent items={data.map(transform)} />

// Good: memoized transformation
const transformedItems = useMemo(() => data.map(transform), [data]);
<MyComponent items={transformedItems} />
```
